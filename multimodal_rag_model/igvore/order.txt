# 1. RAG (Retrieval-Augmented Generation)

**Definition:**
A technique where an LLM first **retrieves relevant information** from an external source and then generates an answer using that information.

**Purpose:**
Prevents the LLM from relying only on what it learned during training.

Example:

```
Question:
"What is the fee structure in this PDF?"

Without RAG:
LLM guesses.

With RAG:
Search PDF → find relevant text → give it to LLM → answer.
```

---

# 2. Embedding

**Definition:**
An embedding is a list of numbers (a vector) that represents the meaning of text or an image.

Example:

```
"The cat is sleeping."

↓

[-0.21, 0.34, ..., 0.87]
```

Your model used **384 numbers** for each text chunk.

**Purpose:**
Makes meaning searchable by converting text/images into numbers.

---

# 3. Embedding Model

**Definition:**
A pretrained AI model that converts text or images into embeddings.

Example:

```
SentenceTransformer - a high level embedding model that does tokenization tensor model normalization and numpy conversion internally
CLIP - low level nmodel needs us to do it
OpenAI Embeddings
```

You used

```
all-MiniLM-L6-v2
```

---

# 4. Vector

**Definition:**
A vector is simply an ordered list of numbers.

Example:

```
[0.2, -0.5, 1.3]
```

An embedding **is a vector**.

---

# 5. Vector Database

**Definition:**
A database that stores vectors and quickly finds the most similar ones.

Example:

```
Chunk 1 → vector

Chunk 2 → vector

Chunk 3 → vector
```

Instead of searching words, it searches meaning.

---

# 6. FAISS

**Definition:**
A vector database/library developed by Meta that performs fast similarity search on embeddings.

You used

```python
faiss.IndexFlatL2()
```

---

# 7. Chunk

**Definition:**
A small piece of a larger document.

Instead of

```
Entire PDF
```

we use

```
Chunk 1

Chunk 2

Chunk 3
```

because LLMs have context limits and searching smaller pieces is more precise.

---

# 8. Chunk Overlap

**Definition:**
Repeating a small part of one chunk in the next chunk.

Example:

```
Chunk 1

The quick brown fox jumps over

Chunk 2

fox jumps over the lazy dog
```

Prevents cutting important information in half.

---

# 9. Similarity Search

**Definition:**
Finding vectors whose meanings are closest to the query vector.

Question

↓

Embedding

↓

Compare with all stored embeddings

↓

Return the closest ones.

---

# 10. L2 Distance

**Definition:**
A mathematical distance between two vectors.

Smaller distance = more similar.

FAISS used

```python
IndexFlatL2
```

to compare embeddings.

---

# 11. Top-K Retrieval

**Definition:**
Return the **K** most relevant chunks.

Example

```
k = 3

↓

Top 3 chunks
```

instead of all 100 chunks.

---

# 16. CLIP

**Definition:**
A pretrained multimodal model from OpenAI that converts **both text and images into the same embedding space**.

Meaning:

```
Dog image

↓

Embedding

Dog sentence

↓

Embedding

↓

Close together
```

This lets you compare text and images directly.
